# Initiative lifecycle

The final kit treats every non-trivial change as a small delivery lifecycle.

## Default lifecycle

1. Bootstrap and refresh project context
2. Read constitution and continuity memory
3. Open or create an initiative directory
4. Write `analysis.md`, `spec.md`, `architecture.md`, `nfr.md`, `plan.md`, `tasks.md`
5. Run artifact consistency and implementation readiness
6. Implement in safe slices
7. Review like an owner
8. Close out the initiative and capture lessons
9. Refresh dashboard and continuity memory

## Why this matters

This flow reduces three common failures of AI-assisted coding:
- code lands before the system is understood
- plans drift from implementation because nobody checks the artifacts together
- future agents repeat the same mistakes because no durable lesson remains
